## 3. The Complete Study System

### Step 1: Create a Claude Project

**Go to:** claude.ai → Projects → Create New Project

- **Project Name:** `TCS NQT 2026 — DSA Preparation`
- **Description:** `Focused DSA preparation for TCS NQT coding section. 2 coding questions, 90 minutes, Python/C++/Java. Target: Digital/Prime roles.`

### Step 2: Upload Your DSA.pdf

Inside the project → Click "Add content" → Upload `DSA.pdf`
This gives Claude persistent access to your topic list and question bank.

### Step 3: Set Project Custom Instructions

Go to Project Settings → Custom Instructions → Paste the following:

```
You are my dedicated DSA tutor helping me prepare for TCS NQT 2026 coding section. Here are your rules:

CONTEXT:
- I am preparing for TCS NQT 2026 (exam in March 2026)
- The coding section has 2 questions, 90 minutes, in C/C++/Java/Python
- I need to clear this for Digital (₹7-7.5 LPA) or Prime (₹9-11 LPA) roles
- My primary language is Python (or specify your language)
- I have uploaded my DSA.pdf with 100+ problems organized by topic and 15 patterns

TEACHING STYLE:
- Always explain the INTUITION first before showing code
- Use simple language; explain like I'm a beginner
- Show the brute force approach first, then optimize
- Always mention Time Complexity (TC) and Space Complexity (SC)
- Give TCS-specific tips (e.g., "TCS often asks to implement sorting manually, don't use .sort()")
- When I share a problem, follow this structure:
  1. Problem Understanding (restate in simple words)
  2. Key Observations & Patterns (which of my 15 patterns applies?)
  3. Approach 1: Brute Force (with TC/SC)
  4. Approach 2: Optimal (with TC/SC)
  5. Clean Code (with comments)
  6. Edge Cases to watch
  7. Similar Problems from my DSA.pdf list
  8. Quick Revision One-liner

EXAM STRATEGY:
- TCS NQT coding: Read both questions first, solve easier one first
- Partial marks exist — solve for at least some test cases even if not optimal
- Manual sort implementations are frequently asked
- String parsing without built-in functions is a TCS favorite
- Number theory (primes, GCD, digit manipulation) appears in almost every exam

PROGRESS TRACKING:
- Keep track of which topics I've covered
- At the end of each session, give me a brief summary of what I learned
- Suggest what to study next based on priority and remaining time
```

## 4. Master Prompts Library

### 🟢 Starting a Study Session
```
I'm starting my DSA study session for today. Based on my DSA.pdf and the 
schedule, today I should study [TOPIC]. Let's begin with the fundamentals 
and work through the problems. Remember to use my 15 patterns framework.
```

### 🔵 Learning a New Topic
```
Teach me [TOPIC] from scratch for TCS NQT coding. Cover:
1. What is it and when to use it
2. The template/pattern code
3. Walk me through the easiest problem first
4. Then an intermediate one
5. Give me one to try on my own
Use Python. Explain like I'm a beginner.
```

### 🟡 Solving a Specific Problem
```
Help me solve LeetCode [NUMBER] - [PROBLEM NAME].
- Which of my 15 patterns does this use?
- Show brute force first, then optimal
- Python code with comments
- What edge cases should I watch for?
- What similar problems from my DSA.pdf should I also practice?
```

### 🔴 Debugging My Code
```
My code for [PROBLEM] is giving wrong output. Here's my code:
```python
[PASTE YOUR CODE]
```
Input: [INPUT]
Expected: [EXPECTED OUTPUT]
Got: [ACTUAL OUTPUT]

Help me find the bug. Don't give the full solution — just point me to 
where I went wrong and give a hint.
```

### 🟣 Pattern Review
```
Give me a quick review of the [PATTERN NAME] pattern:
1. When to recognize it in a problem
2. Template code in Python
3. 3 trigger phrases in problem statements that hint at this pattern
4. Top 5 problems from my DSA.pdf list for this pattern
5. Common mistakes to avoid
```

### ⚫ Quick Revision (Before Exam)
```
Give me a rapid-fire revision of all 15 DSA patterns from my list:
For each pattern, give me ONLY:
- One-line description
- Template code (3-5 lines max)
- One key problem as example
Format as a concise table.
```

### 🟤 Mock Test
```
Give me a TCS NQT style mock coding test:
- 2 problems, similar to actual TCS NQT difficulty
- Include input/output format and constraints
- I'll try to solve them in 90 minutes
- After I submit, grade me out of 10 each
```

### 🔶 Topic-Specific Deep Dives
```
I'm weak at [TOPIC]. Give me:
1. The core concept in simple words
2. The most common problem types (3-4)
3. Step-by-step walkthrough of each type
4. Common traps and edge cases
5. 5 practice problems ranked easy → hard from my DSA.pdf
6. A "cheat code" — the fastest way to solve these in an exam
```

### 🔷 End-of-Session Review
```
Session review time. Today I studied [TOPIC/PROBLEMS]. 
Give me:
1. What I learned today (summary)
2. What I need to revisit
3. What to study tomorrow
4. A quick 3-question quiz on today's topic
5. Confidence rating for this topic
```

### 🟠 TCS-Specific Coding Tips
```
What are the most common TCS NQT coding patterns and traps? Give me:
1. Topics that appear in EVERY TCS NQT exam
2. Things TCS specifically asks (manual sorts, no library functions, etc.)
3. Input/output format tips for TCS
4. How to get partial marks even if I can't solve fully
